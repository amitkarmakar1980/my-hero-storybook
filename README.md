# Hero Storybook

A personalized AI-generated children's storybook app. Upload a photo of your child, pick a theme, and get a fully illustrated 6-page story with your child as the hero.

## Features

- **Personalized stories** — AI writes a 6-page storybook starring your child by name
- **Photo-based character** — upload a child photo; Gemini analyzes it to build a consistent character profile used across every illustration
- **AI illustrations** — full-color storybook images generated by Gemini 3.1 Flash Image
- **6 story themes** — Space Explorer, Ocean Adventure, Jungle Safari, Dragon Tamer, Time Traveler, Magic Forest (3 themes require sign-in)
- **Google Sign-In** — NextAuth v5 with Google OAuth
- **Profile page** — saved stories and uploaded child photos, accessible after sign-in
- **Auto-save** — generated stories and images are saved automatically to Supabase after generation
- **Saved story view** — click any saved story to revisit the full illustrated storybook

## Tech Stack

| Layer | Technology |
|---|---|
| Framework | Next.js 15 (App Router) |
| Auth | NextAuth v5 (Google OAuth) |
| Database | Supabase PostgreSQL via Prisma 7 |
| ORM | Prisma 7 with `@prisma/adapter-pg` |
| Storage | Supabase Storage (story images + child photos) |
| AI — Story | Google Gemini 2.5 Flash (text generation) |
| AI — Images | Google Gemini 3.1 Flash Image Preview |
| Styling | Tailwind CSS v4 |
| Deployment | Vercel |

## Getting Started

### Prerequisites

- Node.js 20+
- A Google Cloud project with OAuth 2.0 credentials
- A Supabase project with two public storage buckets: `story-images` and `child-photos`
- A Gemini API key from [Google AI Studio](https://aistudio.google.com)

### Environment Variables

Create `Code/web/.env.local`:

```env
# Gemini
GEMINI_API_KEY=...
GEMINI_TEXT_MODEL=gemini-2.5-flash
GEMINI_IMAGE_MODEL=gemini-3.1-flash-image-preview

# Auth (NextAuth v5)
NEXTAUTH_SECRET=...
GOOGLE_CLIENT_ID=...
GOOGLE_CLIENT_SECRET=...

# Database
DATABASE_URL=postgresql://...?pgbouncer=true   # port 6543, pooled
DIRECT_URL=postgresql://...                     # port 5432, direct (for migrations)

# Supabase Storage
NEXT_PUBLIC_SUPABASE_URL=https://<project>.supabase.co
SUPABASE_SERVICE_ROLE_KEY=...
```

### Install & Run

```bash
cd Code/web
npm install
npx prisma migrate dev   # sets up database tables
npm run dev              # starts at http://localhost:3000
```

### Google OAuth Setup

1. Go to [Google Cloud Console](https://console.cloud.google.com) → APIs & Services → Credentials
2. Create an OAuth 2.0 Client ID (type: **Web application**)
3. Add authorized redirect URI: `http://localhost:3000/api/auth/callback/google`
4. For production add: `https://yourdomain.com/api/auth/callback/google`
5. If your app is in **Testing** mode, add your email as a test user under the OAuth consent screen

```
    ├── app/

    │   │   ├── generate-story-images/ # Gemini image generation

    │   │   └── profile/photos/       # Fetch uploaded child photos

    │   ├── story-preview/            # Generated story view + auto-save
    │   ├── story/[id]/               # Saved story view
    │   └── profile/                  # User profile (saved stories + photos)
    └── lib/
        ├── auth.ts                   # Full NextAuth config (Prisma adapter)
        ├── auth.config.ts            # Edge-safe NextAuth config (middleware only)
        ├── prisma.ts                 # Prisma client singleton
        ├── prompts.ts                # All Gemini prompts (story + image)
        └── supabase-server.ts        # Supabase server client + upload helpers
```

## Database Schema

Six tables managed by Prisma:

- **User** — NextAuth user record
- **Account / Session / VerificationToken** — NextAuth adapter tables
- **Story** — saved generated story with images (JSON), linked to user
- **UploadedPhoto** — uploaded child photos linked to user

## Deployment

The app is deployed to Vercel. Set all environment variables in the Vercel project settings. The `DIRECT_URL` is used for Prisma migrations only and is not needed at runtime.
